# Attribution Implementation Matrix

This matrix summarizes the current attribution and explainability surface across
`frtb-capital`. It is a navigation aid for reviewers and agents; package-owned
`ATTRIBUTION.md` files remain the source of detailed method semantics, branch
limitations, and test evidence.

Capital attribution is post-calculation. The entries below describe explain
helpers over completed capital results or storage/orchestration projections. They
do not replace regulatory formula documentation or paragraph-level citations in
capital kernels.

## Usage Notes

- Use this matrix to find the owning package and public helper before changing
  attribution behavior.
- Use the package guide linked in the first column for the exact branch rules,
  citations, and limitations.
- Keep `attribution_status` in `docs/quality/package_maturity.toml` aligned
  with this matrix and the [attribution maturity gates](quality/attribution_maturity.md).
- Keep finite-difference impact separate from marginal or standalone attribution.
- Do not close an unsupported branch by adding silent pro-rata allocation; either
  implement a defensible method or keep an explicit unsupported/residual record.
